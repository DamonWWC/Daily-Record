数组:

2. C++11初始化可以省略等号 例如：int arr[5]  {1, 2, 3, 4, 5};
3. 列表禁止缩窄转换。（注：如把int强制转换为short就叫做缩窄转换）

指针：
1. 地址运算符（&）
2. 面向对象编程与传统的过程性编程的区别在于，OOP强调的是在运行阶段（而不是编译阶段）进行决策。运行阶段指是程序正在运行时，编辑阶段指的是编译器将程序组合起来时。
3. 数组和指针，可以修改指针的值，而数组名是常量。
4. 对数组应用sizeof运算符得到的是数组的长度，而指针应用得到的是指针的长度。
5. 数组名被解释为其第一个元素的地址，而对数组名应用地址运算符时，得到的是整个数组的地址，例如：short tell[10] 数组名tell等价于&tell[0]，是一个2个字节内存块的地址，而&tell是一个20字节内存块的地址。
6. C++将数组名视为数组的第一个元素的地址，但是将sizeof运算符用于数组名时，此时将返回整个数组的长度。
7. 指针的数组，例如int* parr[10]，表示parr是一个数组，数组中的每个元素都是一个指向int的指针。array of pointers
8. 数组的指针，例如int (*parr)[10]，表示parr是一个指针，指向一个包含10个int元素的数组。a pointer to an array
9. const pointer，例如int * const parr，parr是一个常量指针，不能修改parr的值，但是可以修改parr所指向的值。
10. pointer to const, 例如const int * parr，parr是一个指针，指向一个int常量，不能修改parr所指向的值，但是可以修改parr的值。
11. 针对9和10两点，关于const修饰的部分，1.看左侧最近的部分，左侧是什么，就修饰什么2.如果左侧没有，则看右侧。
12. 指向指针的指针，例如int ** parr，parr是一个指针，指向一个指针，指向一个int。*操作符具有从右向左的结合性，**表达式相当于*（*parr）,必须从里向外逐层求值。注：用指针进行间接访问之前，确保它已经初始化，并被恰当的赋值。
13. NULL指针，例如int * parr = NULL，parr是一个指针，指向一个int，但是它没有指向任何有效的地址。注：在对一个指针进行间接引用前，请先判断这个指针的值是否为NULL;
14. 未初始化和非法的指针，在C++中，未初始化的指针可能指向任何地方，而非法的指针通常指向内存单元0，这个位置没有存储数据，因此对非法指针进行解引用是非法的。例子：int *a;*a=12;会定位到一个非法地址，程序会出错，从而终止。最坏的情况是定位到一个可以访问的地址，无意修改了它，这样的错误很难被发现。
15. 杜绝野指针，三种情况：1.指针变量没有初始化，2.已经释放不用的指针没有设置NULL,如delete和free之后的指针；3.指针操作超越了变量的租用范围。注：没有初始化的，不用的或者超出范围的指针请把值设置为NULL。
16. 资源管理方案RAII（Resource Acquisition Is Initialization），RAII是一种利用对象生命周期来控制程序资源（如内存、文件句柄、网络连接、互斥量等等）的简单技术。依托栈和析构函数的特性，RAII可以确保在程序运行期间，所有资源都能被正确地释放。比较成熟的智能指针代表：auto_ptr，shared_ptr，weak_ptr，unique_ptr。
17. unique_ptr是专属所有权，只能被一个对象持有，不支持复制和赋值。转移语义：它是禁止了拷贝语义，但是有时需要能转移所有权，于是提供了移动语义，即可以使用move()进行控制所有权的转移。
18. shared_ptr是共享所有权，可以有多个对象持有，支持复制和赋值。引用计数机制，每次有新引用的时候，就加一。引用计数可能会带来个循环引用的问题，也就是死锁。解决方法：使用weak_ptr，它不增加引用计数，但是可以观察资源是否存在。
19. weak_ptr，它不增加引用计数，但是可以观察资源是否存在。
20. 引用，特殊的指针，不能修改的指针，不允许更改指向的指针。不存在空引用，会避免野指针，必须初始化，一旦指向一个地方，就一直引用不会变。可以认为是变量的别名。例如 int& a = b;为了支持函数的运算符重载。


STL(algorithm(算法)、container(容器)、iterator(迭代器))
1. 容器：vector，list，deque，set，map，multiset，multimap，stack，queue，priority_queue。
priority_queue：优先队列，是一种数据结构，它允许以任意顺序插入元素，但是只能检索最大（或最小）元素。
2. 仿函数：一般不会单独使用，主要是为了搭配STL算法使用。
3. 迭代器（iterator）：迭代器是一种检查容器内元素并遍历元素的数据类型。
4. 容器适配器（adapter）：容器适配器是容器的包装器，它将容器的功能扩展到更广的范围。stack，queue，priority_queue。




内存：C++有3种管理数据内存的方式
1. 自动储存：在函数内部定义的常规变量使用自动存储空间，意味着在所属的函数被调用时自动产生，在函数结束时消亡。自动变量通常存储在栈中。
2. 静态存储：整个程序执行期间都存在的存储方式。
3. 动态存储
